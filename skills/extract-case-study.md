# Skill: extract-case-study

> **미팅 기록 1건 → `도입사례_data` 시트 1행 추출·적재** 반복 작업 캡슐.
> 페르소나 [Agents.md](../Agents.md), 규칙 저장소 [Memory.md](../Memory.md)와 함께 사용.

## 언제 쓰나

`callabo-sales-meeting`(또는 담당자 채널)에 세일즈/도입 검토 **미팅 기록**(CSV·텍스트·이미지)이 올라오거나 사용자가 미팅 파일을 첨부할 때.

## 입력

미팅 기록 1건 (예: "유니클로 리스닝마인드 솔루션 도입 검토 미팅.csv").

## 열 스키마

| 열 | 이름 | 추출 규칙 |
|---|---|---|
| A | 산업군 | 분류 표 적용 (아래) |
| B | 플랜 | **확정된 플랜명만.** 미확정 → 공란 |
| C | 기업명 | 미팅 대상 기업 |
| D | 부서 | 도입/활용 주체 부서 |
| E | 도입 사유 / 후킹 포인트 / 활용 방법 | 미팅에서 논의된 내용만 요약 |

### 산업군 분류 표 (→ [Memory.md](../Memory.md)에서 최신 유지)

| 기업/브랜드 유형 | 산업군 |
|---|---|
| 의류/어패럴/스포츠 브랜드 (유니클로·데상트·엄브로·르꼬끄 등) | 생활/소비재 |

> 표에 없는 유형이면 사용자에게 확인 후 표에 추가.

## 절차

1. **읽기** — 첨부 파일/미팅 내용을 읽는다. (CSV·텍스트는 그대로, 이미지는 내용 파악.)
2. **시트 상태 확인** — `도입사례_data` 시트를 조회해 기존 행과 **기업명 중복 여부**를 확인한다. 중복이면 사용자에게 새 행 생성 여부를 묻는다.
3. **추출** — 5개 열을 순서대로 채운다.
   - 산업군: 분류 표 적용
   - 플랜: **명시적으로 도입/플랜이 확정된 경우만** 기입, 아니면 공란
   - 기업명 · 부서: 미팅에서 확인
   - E열: 도입 사유·핵심 후킹 포인트·현재/예정 활용을 요약(미팅에 언급된 것만)
4. **공란·확인** — 근거 없는 셀은 공란. 애매한 셀(특히 플랜)은 사용자에게 확인 질문.
5. **append** — 시트에 **1행만** 추가한다. (기존 행 수정·삭제 금지.)
6. **요약 응답** — Slack에 무엇을 어느 셀에 넣었는지 정리하고, **공란 처리한 셀과 사유**를 고지한다. 현재 총 데이터 행 수를 함께 알린다.

## 완성 예시 (유니클로)

**입력**: "유니클로 리스닝마인드 솔루션 도입 검토 미팅" 기록.

**판단**: 도입/플랜은 확정 안 됨(스탠다드↔어드밴스 갭으로 프로페셔널 후보 언급됐으나 리더·대표 보고 후로 보류) → **B열 공란**.

**적재된 행**:

| 산업군 | 플랜 | 기업명 | 부서 | 도입 사유 / 후킹 포인트 / 활용 방법 |
|---|---|---|---|---|
| 생활/소비재 | *(공란)* | 유니클로 | 마케팅 | 에어리즘 등 핵심 카테고리를 어떻게 더 살릴지가 과제. 검색 시퀀스(패스파인더)로 에어리즘 유입·이탈 경쟁사(무신사 쿨탠다드·탑텐) 파악, 인텐트파인더로 키즈·연관 제품 시장 볼륨 비교, 코파일럿/제미나이 연동 LLM 인사이트·대시보드 자동화에 후킹. 계정 공유·국가별 데이터·과거 비교 모드에 관심. |

**Slack 요약**: 회의록 정리해 1행 추가. 플랜은 근거가 약해 비워뒀는데, 프로페셔널로 채워둘까요? 현재 총 데이터 행 수 N개.
