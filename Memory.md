## 기록 형식

```
- [상황] → [배운 것] → [적용 방법]
```

## 1. 플랜(B열) 판정 규칙

- **플랜 미확정을 억지로 채웠다 → 시트가 사실과 어긋난다 → 도입/플랜이 명시적으로 확정된 경우만 기입, 아니면 공란 + 사용자 확인.**
- 유니클로 사례: 스탠다드↔어드밴스 갭 때문에 신설 "프로페셔널" 플랜이 후보로 언급됐지만 리더·대표 보고 후로 결정이 보류됨 → **플랜 공란 처리**, "프로페셔널로 채울까요?"라고 확인 질문.

## 2. 산업군(A열) 분류 규칙

- **산업군을 즉흥으로 지으면 조회가 흐트러진다 → 정해진 매핑 표를 쓰고, 없는 케이스는 확인 후 표에 추가한다.**

| 기업/브랜드 유형 | 산업군 |
|---|---|
| 의류/어패럴/스포츠 브랜드 (유니클로, 데상트, 엄브로, 르꼬끄 등) | 생활/소비재 |

> 새 산업군이 필요하면 사용자에게 확인 후 이 표에 한 줄 추가.

## 3. 중복 기업 처리

- **같은 기업을 다시 append 하면 시트에 중복 행이 생긴다 → append 전 기업명(C열)이 이미 있는지 확인하고, 있으면 사용자에게 새 행 생성 여부를 묻는다.**

## 4. 도입 사유/후킹/활용(E열) 작성

- **미팅에 없는 활용 방법을 추론해 넣으면 사실과 어긋난다 → E열은 미팅에서 명시·논의된 내용만 요약한다.** (검색 시퀀스·경쟁사 분석·LLM 연동·계정 공유·국가별 데이터 등 실제 언급된 후킹 포인트 위주.)
